# AGENTS.md

本文件只定义当前仓库的协作约束。

## 仓库目标

- 这是一个单机上的 lite LLM 预训练实验仓库
- 默认优先本地可运行，不为分布式场景提前设计
- 以最小闭环为目标，不做与当前训练目标无关的工程扩展

## 修改约束

- 只添加和本仓库当前阶段直接相关的代码、脚本和文档
- 代码和文档修改只允许发生在当前仓库目录内
- 如需临时文件，可使用操作系统的 `temp` 目录
- 优先保持实现简单，避免提前拆分复杂模块
- 没有真实需求时，不引入重型依赖、服务化组件或数据库
- 文档保持精简，避免写与仓库现状不符的说明

## 执行原则

- 新增命令时，优先保证在当前电脑上可以直接运行
- 新增 README 内容时，只描述已经存在或当前准备落地的能力
- 做结构调整时，优先服务于数据、分词、训练、评估这条主链路

## 协作记录

- 仓库根目录的 `PROGRESS.md` 是唯一的进度与尝试记录入口
- 开始一项实质工作前，先在 `PROGRESS.md` 更新当前任务和计划动作
- 每次完成一次重要迭代后，在 `PROGRESS.md` 的 `Recent Log` 顶部追加记录
- 任务推进中如遇阻塞、风险或关键决策，及时更新 `PROGRESS.md`
- 完成一次提交前，更新 `PROGRESS.md` 中的当前状态、最新进展和下一步
- `PROGRESS.md` 只保留当前上下文和最近少量关键记录，避免文件持续膨胀
- 更早的记录按月归档到 `progress/archive/YYYY-MM.md`
- 工作记录优先记录目标、改动、验证、问题、结论和下一步，避免维护重复模板

## 提交原则

- 每次改动先执行必要的检查或验证，确认通过后再提交
- 验证通过后，可直接提交并推送，无需额外确认
- 单个 commit 保持尽量小，只解决一个明确问题
- commit 需要可回滚、可追溯、可 review，避免混入无关改动
